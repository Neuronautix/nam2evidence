# Security Policy

## Supported Versions

The public `main` branch is the supported development line for security reports.

## Reporting a Vulnerability

Please do not disclose security vulnerabilities in public issues.

Use GitHub private vulnerability reporting if it is enabled for this repository.
If it is not enabled, contact the repository maintainers privately and include:

- A clear description of the vulnerability.
- Steps to reproduce.
- Affected components and versions or commit hashes.
- Any proof-of-concept details that are safe to share privately.

## Scope

Security reports are welcome for application code, Docker configuration, API
behavior, dependency exposure, and demo-data handling. Please do not submit
reports for intentionally public documentation, demo-only placeholder data, or
known limitations already documented as proof-of-concept constraints.
