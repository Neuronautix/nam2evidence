Project Overview Tab
====================

The Overview tab is the demo front door. It presents the before-to-after story
as a standardization status dashboard rather than a generic project summary.

Hero status
-----------

The top hero foregrounds:

* the one-line before/after standardization pitch;
* Context of Use confidence and project review status;
* the FAIR / AI-readiness percentage;
* the reminder that readiness is a proof-of-concept score requiring qualified
  human review.

Before to after panel
---------------------

This panel makes the core transformation explicit:

Before:
  Spreadsheet rows, protocol notes, free-text terms, and unreviewed regulatory
  claims.

After:
  Canonical endpoint measurements, approved ontology links, validation blockers,
  provenance, readiness, and reusable exports.

Standardization workflow
------------------------

The workflow cards lead the live demo through:

* CSV import;
* column mapping;
* ontology approval;
* validation gates;
* readiness;
* provenance.

Status metrics
--------------

The metric cards are clickable and summarize:

* endpoint rows standardized;
* ontology mappings approved;
* semantic blockers;
* human-reviewed claims.

Top blockers
------------

The blocker panel pulls from semantic validation, ontology state, and claim
review state. Each blocker links to the workspace where it can be inspected or
resolved.

Reusable package outputs
------------------------

The output panel shows the payoff formats:

* JSON-LD;
* RDF/Turtle;
* RO-Crate;
* ISA-Tab;
* Parquet;
* eCTD map.

How to use it
-------------

Use Overview as the control room. Start with readiness and blockers, then follow
the workflow cards from Endpoint Data through Provenance before using the
downstream Context of Use, Claim Graph, eCTD, Audit, and Export screens.

What good looks like
--------------------

Before export, you should see:

* endpoint rows imported and standardized;
* mandatory ontology mappings approved;
* no semantic blockers;
* readiness gaps understood or resolved;
* provenance traced for endpoint values;
* claims approved by a qualified reviewer;
* context-of-use limitations visible;
* exports downloaded only after review gates allow it.
