Audit Tab
=========

The Audit tab shows a reverse-chronological record of changes across NAM-CORE
workspaces.

Purpose
-------

The audit trail supports transparency. It helps reviewers understand what
changed, when it changed, and why.

Columns
-------

The table includes:

* timestamp;
* action;
* entity type;
* entity ID;
* user or role;
* reason.

How to use it
-------------

Use the audit trail when:

* checking who approved mappings;
* understanding why a value changed;
* reviewing export history in combination with the Export Center;
* investigating unexpected validation changes.

Limitations
-----------

The current proof-of-concept audit trail is not a validated 21 CFR Part 11
system. Treat it as transparency support, not a compliant electronic records
system.
