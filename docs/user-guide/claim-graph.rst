Claim Graph Tab
===============

The Claim Graph tab shows structured claims and their relationships to evidence.
It is a human review gate.

Purpose
-------

Claims translate study evidence into statements that might appear in an evidence
package or regulatory narrative. Because claims are interpretive, qualified
review is required.

What you see
------------

The graph includes:

* claim IDs;
* claim text;
* confidence level;
* review status;
* parent/child or support relationships.

Review statuses
---------------

Pending or human-review-required claims block export. Approved claims allow the
package to move forward, assuming no other blockers remain.

How to review claims
--------------------

For each claim:

1. Read the exact claim wording.
2. Check whether evidence supports it.
3. Confirm it fits the Context of Use.
4. Approve only if wording, evidence, and limitations are aligned.
5. Reject or leave pending if it overstates the evidence.

Common problems
---------------

Overclaiming:
  The claim is stronger than the data.

Context mismatch:
  The claim may be true for the assay but not for the declared regulatory use.

Missing caveats:
  Known limitations should be reflected in the claim or dossier.
